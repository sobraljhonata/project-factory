# Extraction Criteria

## Classificações
- core: parte sempre presente no boilerplate
- template: parte parametrizável
- optional module: parte opcional selecionável
- remove: parte específica do projeto atual

## O que observar
- nomes específicos do projeto
- URLs, buckets, ARNs, domínios
- variáveis de ambiente acopladas
- workflows específicos
- docs específicas
- recursos de infra não generalizáveis
